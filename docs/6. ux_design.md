아래는 SRS v0.2(REQ/POL/NFR/RTM)와 Strategic Design(BC 경계/관계)을 전제로, 
Release 1 UX/UI 설계를 “화면 IA → 핵심 플로우 → 화면별 스펙(입력/상태/검증) → BC 매핑” 순서로 정리한 초안입니다. 
요구사항의 “기록 우선/나중에 보정”과 자동 생성 신뢰성/리포트 응답성 목표를 UX 의사결정의 최상위 제약으로 둡니다.  

────────────────────────────────────────────────────────
0. UX 설계 원칙(Release 1)
────────────────────────────────────────────────────────
1. 기록 우선, 보정 용이(Quick capture, later refine)
거래 입력은 최소 필드로 즉시 저장되고, 이후 편집에서 분류/태그/첨부를 보완하는 흐름을 기본으로 둡니다. 

2. 점진적 공개(Progressive disclosure)로 입력 부담 최소화
필수 입력만 1차 영역에 두고, 고급 옵션(태그/메모/첨부 등)은 2차 영역(“추가 옵션”)으로 노출한다.

3. 인지 부담 절감: 회상보다 인지(Recognition over recall)
카테고리/태그 선택은 자동완성·최근 사용·추천(최근 5개) 등 “보여주고 고르는” 방식으로 설계합니다.

4. 상위 목적 3~5개 내비게이션 고정
모바일 기준 하단 내비게이션(3~5 목적지)을 사용하고, “거래 추가”는 내비게이션이 아닌 별도 Primary Action(예: FAB/고정 버튼)으로 분리합니다.

1. UX 결정사항(Release 1)
────────────────────────────────────────────────────────
1.1 화면 형태(PC 포함)
* PC에서도 “화면 전체를 확장하여 데스크톱 레이아웃을 구성”하지 않는다.
* 모바일 앱처럼 세로로 긴 단일 컬럼 UI를 그대로 제공한다.
* 구현 규칙: 중앙 정렬된 고정 폭 컨테이너(max-width) 안에서만 렌더링하며, 좌우 여백은 배경 영역으로 처리한다.

1.2 내비게이션
* 하단 내비게이션 4탭 + 거래 추가 FAB(우측 하단).

1.3 비활성 리소스 처리(UX-001)
* 기존 거래(목록/상세/리포트)에서는 비활성 자산/카테고리/태그를 “표시”한다.
* 신규 생성/수정(입력/편집) 선택지에서는 비활성 항목을 “노출하지 않는다(선택 불가)”.

1.4 첨부(UX-002 변경 반영)
* 첨부는 “이미지 파일만” 허용한다.
* 거래당 첨부 최대 2개(POL-LIMIT-001).
* 이미지 미리보기 제공(썸네일/확대).

1.5 필수 입력 강화(추가 결정)
* 거래 생성(홈 “+ 거래”)에서 1단계 카테고리는 필수 선택이다.
* 고정비 템플릿 생성/편집에서도 1단계 카테고리는 필수 선택이다.
* 주의: SRS의 “카테고리 지정 가능” 요구(REQ-LEDGER-004)를 “필수”로 강화하는 UX 결정이므로, API Spec 단계에서 서버 검증 규칙으로 명시해야 한다. 

1.6 아키텍처 전제
* 모듈러 모놀리스. Bounded Context 경계를 화면 모듈 경계로 매핑한다.

3. 정보구조(IA) 및 BC 매핑
────────────────────────────────────────────────────────
BC 경계는 화면 모듈 경계로 그대로 반영.(모듈러 모놀리스 전제)

3.1 Top-level(3~5 목적지)
* 홈(원장): 거래/이체 목록, 필터, 빠른 입력 진입점
* 리포트: 월간 리포트(자산/카테고리 롤업/태그/예산 대비)
* 계획: 월 예산, 고정비 템플릿(자동 생성 설정/상태)
* 설정: 자산/카테고리/태그 관리, 로그아웃

3.2 화면 목록(Release 1)
A. 인증(IAM)
* 로그인

B. Ledger
* 홈/원장(거래·이체 목록)
* 거래 상세(읽기)
* 거래 추가/편집(지출/수입)
* 이체 추가/편집
* 첨부 미리보기(이미지)

C. Reporting
* 월간 리포트(월 선택 포함)
* 축별 상세(자산/카테고리/태그) + 예산 대비

D. 계획(Budget/Automation)
* 월 예산 설정
* 고정비 템플릿 목록/추가/편집
* 자동 생성 상태/누락 감지 뷰(최소: 배너/상태 리스트)

E. 설정(Classification/Asset)
* 카테고리 관리(기본 카테고리 + 개인화)
* 태그 관리
* 자산 관리

4. 핵심 사용자 플로우(ConOps/RTM 기반)
────────────────────────────────────────────────────────
4.1 거래 기록(수입/지출) — RTM-001
* 홈 FAB “+ 거래” → 거래 추가(필수 입력 완료 시 저장) → 목록 즉시 반영 → 필요 시 상세/편집에서 보완. 
* 필수 입력: 금액, 날짜(기본 오늘), 유형(지출/수입), 자산, 카테고리(1단계)
* 선택 입력: 카테고리(2단계), 태그(≤3), 첨부(≤2), 메모

4.2 이체 기록 — RTM-002
* “+ 거래”에서 유형 = 이체
* 필수 입력: 출금 자산, 입금 자산(동일 불가), 금액, 날짜
* 선택 입력: 메모
* 리포트/예산 비교에는 “이체 제외” 명확 표시(정보 설계로 오해 방지)

4.3 고정비 템플릿 및 자동 생성 — RTM-004
* 계획 > 고정비 템플릿 추가(필수 입력 완료) → 매월 지정일 자동 생성 → 생성 거래는 출처 표시(자동) → 실패/지연 시 누락 감지 UI 제공.  
* 필수 입력: 발생일, 금액, 기본 자산, 카테고리(1단계)
* 선택 입력: 메모, 태그, 카테고리(2단계)

4.4 월간 리포트 — RTM-003/006
* 리포트 탭 → 월 선택(기본 현재월) → KPI/축별 요약(자산/카테고리/태그) + 예산 대비.  

4.5 기본 카테고리 제공/개인화 — RTM-007
* 설정 > 카테고리 관리 → 기본 카테고리 확인 → 수정/비활성화(개인 설정으로 반영) → 조회 시 사용자 관점 목록 제공.

5. 화면별 UX/UI 상세 스펙
────────────────────────────────────────────────────────
5.1 로그인(IAM)
* 입력: 아이디/비밀번호(최소)
* 성공: 홈 이동
* 실패: 공통 에러 토스트 + 필드 하단 메시지

5.2 홈/원장: 거래 목록
목표: “빠른 기록”과 “정리 가능성” 동시 만족(NFR-USAB-001).
구성:
* 상단: 월 선택(드롭다운), 검색/필터(자산/카테고리/태그)
* 본문: 날짜 그룹(일자 헤더) + 거래/이체 아이템(금액, 자산, 카테고리, 태그 chip, 출처 배지(자동))
* FAB: “+ 거래”
상태:
* Empty: “첫 거래를 기록해보세요” + “+ 거래”
* Loading: 스켈레톤
* Error: 재시도 CTA
정책 반영:
* 삭제 처리 거래는 목록/리포트에서 제외(사용자 관점에선 ‘삭제됨’ 노출 불필요) 
* 자동 생성 거래 출처 표시 

5.3 거래 추가/편집(지출/수입)
1차 입력(기본 영역: 반드시 보임)
* 금액(amount): 숫자
* 날짜(date): 기본 오늘
* 유형(type): 지출/수입
* 자산(asset): 드롭다운/검색(최근 사용 우선)
* 1단계 카테고리(categoryLevel1)

2차 입력(접기/펼치기: “추가 옵션”)
* 카테고리 2단계(categoryLevel2): 1단계 선택 후에만 노출(점진 공개)
* 태그(tags): 최대 3개(POL-LIMIT-002) 
* 첨부(attachments): 이미지 최대 2개(POL-LIMIT-001) 
* 메모(memo)

비활성 항목(카테고리, 자산, 태그) 처리
* 생성: 비활성 항목은 선택 리스트에서 제외(POL-CLASS-RES-001). 
* 편집:
  * “기존 거래에 이미 연결돼 있던 항목이 이후 비활성화된 경우” 표시/유지 허용(읽기)
  * 변경하려면 활성 항목만 선택 가능(드롭다운에 비활성은 없음)
  * “현재 비활성 값을 유지한 채 저장”은 허용(기존 거래 표시 정책과 일관)

첨부 UX(이미지 전용)
* “이미지 추가” 버튼 → 파일 선택(이미지 MIME/확장자 화이트리스트는 API Spec에서 고정)
* 썸네일 2칸 그리드(최대 2)
* 썸네일 클릭: 확대 미리보기(라이트박스/모달)
* 업로드 실패:
  * 거래 저장은 성공 유지(REQ-LEDGER-007 취지)
  * 썸네일 위치에 “업로드 실패/재시도” 표시 + 재시도 버튼 

제약 위반 처리
* 태그 4번째 선택 시 즉시 차단 + 인라인 안내
* 첨부 3번째 추가 시 즉시 차단 + 인라인 안내
* 서버 검증 실패 시(경합/동시성 포함): 폼 상단 공통 에러 + 해당 필드 인라인 에러

5.4 이체 추가/편집
* 필드: 출금 자산(fromAsset), 입금 자산(toAsset), 금액, 날짜, 메모
* 검증: fromAsset != toAsset (즉시 차단) 
* 리포트에는 “자산 이동(이체)”로 분리 표기(소비 오해 방지) 

5.5 리포트(월간)
목표: Reporting Core 경험(다축 회고 + 예산 대비)
구성
* 월 선택(기본 현재월)
* 상단 KPI: 총지출, 총수입, 순지출, 예산 사용률/잔액(예산 미설정 시 CTA)
* 섹션1: 자산별 요약(리스트/간단 바)
* 섹션2: 카테고리(대분류 롤업) 요약 + “상세 보기”(드릴다운은 2-depth로)
* 섹션3: 태그별 지출 합계
응답성
* 첫 진입 시 스켈레톤, 재조회(월 변경) 시 “부분 로딩”으로 체감 지연 최소화 

5.6 계획: 월 예산
* 월 선택 → 금액 설정/수정/해제

5.7 계획: 고정비 템플릿(Automation)
* 목록: 템플릿명, 발생일, 금액, 활성/비활성
* 생성: 발생일(dayOfMonth), 금액, 자산, 1단계 카테고리, 태그(선택), 메모(선택)
* 정책: 동일 월 중복 생성 방지(사용자에겐 “이미 이번 달 생성됨” 정도로 안내) 
* 누락 감지: “이번 달 생성 실패/지연 가능” 배너 + 상세 리스트(최소)
