Product Vision (가계부 서비스) v0.5

1. 제품 한 줄 정의(One-liner)
개인 사용자가 수입/지출/이체를 수동으로 기록하고, 자산(원천)·카테고리·태그로 구조화하여 월별 리포트 및 예산 대비로 소비 패턴을 해석할 수 있는 가계부 서비스.

2. 타깃 사용자(Target)
* 개인 사용자(가구/공유 단위 제외)
* 통화: KRW 고정
* 입력 방식: 수동 입력
* 주요 사용 주기: 결제 직후 기록, 주 1회 정리, 월 1회 리포트/예산 대비 확인

3. 해결하려는 핵심 문제(Problems)
* 기록 지속성 부족: 입력 피로로 누락/사후 정리 비용 증가
* 회고 어려움: 카테고리만으로 목적/상황/프로젝트 단위 분석 한계
* 원천 추적 부족: 카드/현금/계좌 등 지출 발생 원천 파악의 비일관성
* 계획 대비 비교 어려움: 월 예산 대비 실적을 빠르게 확인하기 어려움
* 반복 입력 부담: 고정비 등 반복 지출의 입력 비용
* 증빙 분리: 영수증 등 첨부가 거래와 분리되어 회고 비용 증가

4. 제품 가치 제안(Value Proposition)
* 빠른 기록으로 누락 감소
* 자산/카테고리/태그 다축 분류로 리포트 해석력 강화
* 예산은 “통제”가 아닌 “비교” 기준으로 제공해 진입장벽 최소화
* 고정비 자동 생성으로 반복 입력 비용 절감
* 첨부(영수증 이미지 등)를 거래에 결합해 회고 비용 절감

5. 차별점(Differentiators)
* 다축 분석: 자산(원천)·카테고리(구조)·태그(횡단) 기반 리포트
* 예산의 포지셔닝: 통제/차단이 아닌 계획 대비 비교에 집중
* 반복 지출 자동화: 고정비 자동 생성으로 기록 누락을 구조적으로 감소
* 증빙 결합: 거래-첨부를 함께 관리해 회고 가능성 강화

6. Release 1 범위(In-Scope, 기능군 수준)
* 거래 기록: 수입/지출/이체의 등록/관리
* 분류: 카테고리·태그 기반 분류 및 필터/집계
* 자산: 카드/현금/은행계좌 기반 “원천 추적”
* 예산: 월 예산을 기준으로 계획 대비 리포트 제공
* 고정비: 반복 거래 자동 생성
* 첨부: 거래에 대한 파일(영수증 이미지 등) 업로드/관리
* 리포트: 월간 리포트(카테고리/태그/자산/예산 대비)

7. Release 1 비범위(Out-of-Scope, 스코프 상한선)
* 자동 수집(오픈뱅킹/카드 연동), OCR 영수증 인식
* 자산 잔액/정산/카드 청구 사이클 모델링
* 예산 통제(초과 차단/강제), 고급 알림/추천/예측
* 금융권 휴일 조정 및 복잡 반복 규칙
* 환불/취소 모델 및 원거래 연결
* 가구 공유/정산, 다중 통화, 외부 내보내기/회계 연동

8. 성공 지표(Success Metrics) - HEART 프레임워크 참고
* Retention(지속 사용):
  * retainedUsers30dRate - 가입일 기준 30일차(±N일) 내 transactionCreated 1회 이상 발생 사용자 / 해당 코호트 사용자
* Engagement(기록 습관/활동 강도): 
  * weeklyEntryDays - 주간 transactionCreated 발생 ‘일수’의 평균(사용자 단위)
  * avgMonthlyTransactionsPerUser - 사용자당 월 transactionCreated 평균 건수
* Adoption(핵심 기능/구조화 채택): 
  * assetAssignedTransactionRate - 자산이 지정된 거래 / 전체 거래
  * taggedTransactionRate - 태그 1개 이상 연결된 거래 / 전체 거래
  * fixedCostTemplateSetupRate - fixedCostTemplate 1개 이상 생성 사용자 / 신규 활성 사용자(예: 가입 7일 이내 활성)
* Task Success(핵심 과업 성공/효율/오류): 
  * transactionCreateSuccessRate - 거래 생성 성공 / 거래 생성 시도(에러 이벤트 기반)
  * autoGenFailureRate: 자동 생성 실패 건수 / 자동 생성 실행 건수(배치/스케줄러 단위)

9. 핵심 리스크(Risks)
* 수동 입력 피로로 인한 이탈(UX 의존)
* 태그 과다 생성으로 인한 데이터 품질/검색 비용 증가
* 고정비 자동 생성의 중복/실패 등 운영 리스크
* 첨부 저장/운영 비용 및 확장성 리스크
